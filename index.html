Data Structure Foundations

This section introduces the core concepts of data organization and algorithms. A data structure is a way of collecting and organizing data to perform operations effectively.

Algorithm Properties

  • ✓Input: 0 or more inputs supplied externally.
  • ✓Output: At least 1 output obtained.
  • ✓Definiteness: Every step must be clear and well-defined.
  • ✓Finiteness: Must have a finite number of steps.
  • ✓Correctness: Must generate a correct output.

Tree Anatomy & Binary Trees

This section breaks down the hierarchical nature of Tree Data Structures. You will explore the terminology used to describe tree components.

Binary Tree Structure

Terminology Glossary

  • Root The topmost node of the tree. There is only one root per tree.
  • Parent & Child A node with an edge downward is a parent. The node below is its child.
  • Leaf A node which does not have any child nodes.
  • Subtree Represents the descendants of a node, forming a smaller tree within.

    Search Algorithms

    This section contrasts sorting and searching methodologies. Algorithms are designed to retrieve an element from a structure. Interval searches (like Binary Search) dramatically reduce search time compared to Sequential searches.

    Linear Search

    Sequential check over all items one by one. Time complexity increases linearly with data size.

    Binary Search

    Divide and conquer rule. Mandatory: Target array must be sorted. Halves the searchable items each step.
