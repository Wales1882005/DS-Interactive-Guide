Tree Anatomy & Binary Trees

This section breaks down the hierarchical nature of Tree Data Structures. You will explore the terminology used to describe tree components.

Binary Tree Structure

Terminology Glossary

  • Root The topmost node of the tree. There is only one root per tree.
  • Parent & Child A node with an edge downward is a parent. The node below is its child.
  • Leaf A node which does not have any child nodes.
  • Subtree Represents the descendants of a node, forming a smaller tree within.

Binary Search Tree (BST) Explanation

A Binary Search Tree is a special type of binary tree where each node follows an ordering rule: all values in the left subtree are smaller than the current node, and all values in the right subtree are greater. This structure makes searching, insertion, and deletion efficient on average.

BST Rule

  • Left child < Parent
  • Right child > Parent
  • This rule applies recursively

Why It Matters

  • Faster lookup than linear scan
  • Keeps data ordered naturally
  • Supports dynamic updates

Traversal Tip

In-order traversal (Left, Node, Right) of a BST returns values in ascending order.

Example: If the root is 50, then 30 can be placed on the left and 70 on the right. If you insert 65, it goes to the right subtree of 50, then to the left of 70.

    Expression Parsing & Spanning Trees

    This section details how arithmetic expressions are represented and evaluated by computers using Stacks and Expression Trees, as well as the concept of Spanning Trees.

    How Infix, Prefix, and Postfix Work

    Expression parsing is the process of converting and evaluating arithmetic expressions in a format that computers can process efficiently. Infix is human-friendly, while Prefix and Postfix are easier for machines and stack-based evaluation.

    Infix

    Operator is placed between operands.

    A + B * C

    Prefix (Polish)

    Operator is placed before operands.

    + A * B C

    Postfix (Reverse Polish)

    Operator is placed after operands.

    A B C * +
    Why Postfix is common in stacks: Postfix removes the need for parentheses and operator precedence checks during evaluation. You can process tokens from left to right using a stack: push operands, pop/apply on operators, then push result.

    Postfix Evaluation Algorithm (Stack Walkthrough)

    Evaluating: 6 4 2 + - 3 8 16 / + * 2 ^ 3 +

    Token Action Stack State (Bottom → Top) Calculation

    4Push[6, 4]-

    +Pop 2, Eval, Push[6, 6]4 + 2 = 6
    -Pop 2, Eval, Push[0]6 - 6 = 0
    3Push[0, 3]-
    8Push[0, 3, 8]-
    16Push[0, 3, 8, 16]-
    /Pop 2, Eval, Push[0, 3, 2]16 / 8 = 2
