Data Structure Foundations

This section introduces the core concepts of data organization and algorithms. A data structure is a way of collecting and organizing data to perform operations effectively. Algorithms are the finite sets of instructions used to manipulate them.

Algorithm Properties

  • ✓Input: 0 or more inputs supplied externally.
  • ✓Output: At least 1 output obtained.
  • ✓Definiteness: Every step must be clear and well-defined.
  • ✓Finiteness: Must have a finite number of steps.
  • ✓Correctness: Must generate a correct output.

Graphs & Trees

This section explores hierarchical and network-based data structures. Graphs connect objects (vertices) via links (edges). Trees are a specific acyclic subset of graphs defining parent-child relationships.

Graph Concepts

  • Vertex (Node): Represented as interconnected points.
  • Edge (Link): The path or line between two vertices.
  • Directed vs Undirected: One-way vs Two-way connections.
  • Adjacency Matrix: A 2D array representation of edge connections (1 if connected, 0 if not).
  • Path & Cycle: A sequence of edges; a cycle starts and ends at the same node.

Tree & BST Concepts

  • Root & Leaf: The top node vs a node with no children.
  • Binary Search Tree (BST): Left child < Parent ≤ Right child.
  • Spanning Tree: Subset of a graph connecting all vertices with minimum edges (no cycles).
  • Traversals (DFT):
    - In-Order (LNR): Sorts BST ascending.
    - Pre-Order (NLR): Root visited first.
    - Post-Order (LRN): Root visited last.

Search Algorithms

This section contrasts sorting and searching methodologies. Algorithms are designed to retrieve an element from a structure. Interval searches (like Binary Search) dramatically reduce search time compared to Sequential searches by dividing the search space.

Linear Search

Sequential check over all items one by one. Time complexity increases linearly with data size.

1. i = 1
2. if A[i] == x return i
3. i = i + 1, loop

Binary Search

Divide and conquer rule. Mandatory: Target array must be sorted. Halves the searchable items each step.

mid = low + (high - low) / 2

Time Complexity Comparison

Illustrates generic growth: Linear O(n) vs Binary O(log n). Binary search flattens out, handling large datasets efficiently.
